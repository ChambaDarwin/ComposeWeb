Interfaces en Java

Aprende a crear contratos entre clases

¿Qué es una Interfaz?

Una interfaz es una colección de métodos abstractos que establece un contrato con las clases que la implementan. Cuando una clase implementa una interfaz, está obligada a proporcionar una implementación concreta de todos los métodos abstractos definidos en ella.

🔌 Analogía: Piensa en una interfaz como un enchufe eléctrico estándar. El enchufe define la forma y el voltaje que debe tener cualquier dispositivo para conectarse (el contrato), pero no especifica qué hará el dispositivo una vez conectado. Una lámpara, un televisor o una licuadora pueden usar el mismo tipo de enchufe, pero cada uno hace algo completamente diferente. La interfaz solo dice "si quieres conectarte aquí, debes tener estas características", pero tú decides qué hacer con la electricidad.

Es importante entender que la interfaz solo define los métodos, no cómo deben implementarse. Esto significa que distintas clases pueden implementar la misma interfaz y comportarse de manera completamente diferente.

Sintaxis para Crear una Interfaz

Una interfaz se crea utilizando la palabra reservada interface, seguida del nombre de la interfaz. Dentro de las llaves, declaramos los métodos abstractos sin implementación:

interface Operacion {
    // Métodos abstractos (solo declaración, sin cuerpo)
}

Cómo Crear Métodos Abstractos

Los métodos en una interfaz se declaran especificando el tipo de retorno, el nombre del método y los parámetros. Si el método no devuelve ningún valor, utilizamos void. Si devuelve un valor, especificamos el tipo de dato como int, float, String, etc.

interface Operacion {
    int sumar(int a, int b);
    int restar(int a, int b);
    int multiplicar(int a, int b);
    float dividir(int a, int b);
}

⚠️ Importante: En una interfaz, los métodos solo se declaran, NO se implementan. No llevan llaves ni código dentro. La implementación se realiza en la clase que implementa la interfaz.

Implementando la Interfaz

Ahora creamos una clase llamada Matematica que implementará los métodos abstractos de la interfaz. Para ello, utilizamos la palabra reservada implements seguida del nombre de la interfaz:

class Matematica implements Operacion {
    int num1, num2;
    
    // Constructor
    public Matematica(int num1, int num2) {
        this.num1 = num1;
        this.num2 = num2;
    }
    
    // Implementación del método sumar
    public int sumar(int a, int b) {
        return a + b;
    }
    
    // Implementación del método restar
    public int restar(int a, int b) {
        return a - b;
    }
    
    // Implementación del método multiplicar
    public int multiplicar(int a, int b) {
        return a * b;
    }
    
    // Implementación del método dividir
    public float dividir(int a, int b) {
        if (b != 0) {
            return (float) a / b;
        } else {
            System.out.println("Error: No se puede dividir por cero");
            return 0;
        }
    }
}

La clase Matematica está obligada a implementar todos los métodos declarados en la interfaz Operacion. Observa que ahora los métodos tienen llaves y contienen el código que ejecuta la operación correspondiente.

📝 Analogía: Si la interfaz es como un contrato de trabajo, la clase que implementa es como el empleado que firma ese contrato. El contrato dice "debes realizar estas tareas" (los métodos abstractos), pero no especifica exactamente cómo hacerlas. El empleado (la clase) es quien decide la mejor manera de cumplir con cada tarea, siempre y cuando las complete todas.

Usando la Clase que Implementa la Interfaz

Finalmente, creamos un objeto de la clase Matematica y utilizamos sus métodos:

public class Main {
    public static void main(String[] args) {
        // Creamos un objeto de la clase Matematica
        Matematica calculadora = new Matematica(10, 5);
        
        // Usamos los métodos implementados
        System.out.println("Suma: " + calculadora.sumar(10, 5));           // 15
        System.out.println("Resta: " + calculadora.restar(10, 5));         // 5
        System.out.println("Multiplicación: " + calculadora.multiplicar(10, 5)); // 50
        System.out.println("División: " + calculadora.dividir(10, 5));     // 2.0
    }
}

Ventajas de Usar Interfaces

Las interfaces ofrecen múltiples beneficios en la programación orientada a objetos. Permiten crear código más flexible y mantenible, ya que diferentes clases pueden implementar la misma interfaz de formas distintas. Esto facilita el polimorfismo, permite cambiar implementaciones sin afectar el resto del código, y establece un estándar claro sobre qué métodos debe tener una clase.
