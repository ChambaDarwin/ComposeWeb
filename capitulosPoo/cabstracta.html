Clases Abstractas en Java

Aprende a crear jerarquías de objetos relacionados

¿Qué es una Clase Abstracta?

Una clase abstracta se utiliza para representar una jerarquía de objetos relacionados. No se puede instanciar directamente (no puedes crear objetos de ella), pero sirve como plantilla o base para otras clases que heredan de ella.

🏗️ Analogía: Imagina que una clase abstracta es como un plano arquitectónico genérico de "Edificio". Este plano define que todo edificio debe tener ciertos elementos básicos (puertas, ventanas, estructura), pero no especifica todos los detalles. No puedes construir un "edificio genérico" directamente, necesitas especificar si será una casa, un hospital o una escuela. Cada tipo de edificio hereda las características básicas del plano genérico, pero añade sus propias especificaciones. La clase abstracta es ese plano base que dice "todo lo que herede de mí debe tener estos métodos", pero no es útil por sí sola.

Ejemplo Práctico: Sistema de Empleados

Para entender cómo trabajan las clases abstractas, crearemos una clase general llamada Empleado (abstracta) y una clase específica llamada Docente que hereda de ella:

classDiagram class Empleado { <<abstract>> -nombre: String -cedula: String +salario()* float +mostrarInfo() void } class Docente { -horasClase: int -tarifaHora: float +salario() float } class Main { +main(args: String[]) void } Empleado <|-- Docente : hereda Main --> Docente : crea y usa

Creando la Clase Abstracta

Para declarar una clase como abstracta, utilizamos la palabra reservada abstract. Dentro de ella podemos tener métodos normales (con implementación) y métodos abstractos (sin implementación):

abstract class Empleado {
    // Atributos comunes a todos los empleados
    protected String nombre;
    protected String cedula;
    
    // Constructor
    public Empleado(String nombre, String cedula) {
        this.nombre = nombre;
        this.cedula = cedula;
    }
    
    // Método abstracto: cada tipo de empleado calculará su salario
    public abstract float salario();
    
    // Método concreto: todos los empleados pueden usar este método
    public void mostrarInfo() {
        System.out.println("Empleado: " + nombre);
        System.out.println("Cédula: " + cedula);
        System.out.println("Salario: $" + salario());
    }
}

⚠️ Nota importante: Los métodos marcados como abstract no tienen cuerpo (no llevan llaves). Las clases que hereden de Empleado están obligadas a implementar el método salario().

Implementando la Clase Concreta

Ahora creamos la clase Docente que hereda de Empleado. Para heredar, utilizamos la palabra reservada extends:

class Docente extends Empleado {
    private int horasClase;
    private float tarifaHora;
    
    // Constructor
    public Docente(String nombre, String cedula, int horasClase, float tarifaHora) {
        super(nombre, cedula); // Llamamos al constructor de la clase padre
        this.horasClase = horasClase;
        this.tarifaHora = tarifaHora;
    }
    
    // Implementación obligatoria del método abstracto
    @Override
    public float salario() {
        return horasClase * tarifaHora;
    }
}

👨‍🏫 Analogía: Si Empleado es un contrato que dice "todos los empleados deben poder calcular su salario", entonces Docente es una persona específica que firma ese contrato y explica exactamente cómo calculará su salario (horas × tarifa). Otros tipos de empleados podrían calcular su salario de manera diferente (salario fijo mensual, por comisiones, etc.).

Uso de la Clase Concreta

Ahora podemos crear objetos de la clase Docente y utilizar tanto los métodos heredados como los propios:

public class Main {
    public static void main(String[] args) {
        // Creamos un objeto de tipo Docente
        Docente docente = new Docente("María González", "1234567890", 120, 15.5f);
        
        // Usamos el método implementado
        System.out.println("Salario del docente: $" + docente.salario());
        
        // También podemos usar el método heredado
        docente.mostrarInfo();
        
        // ❌ ESTO NO ES POSIBLE:
        // Empleado emp = new Empleado("Juan", "9876543210");
        // No podemos instanciar directamente una clase abstracta
    }
}

La salida del programa sería:

Salario del docente: $1860.0
Empleado: María González
Cédula: 1234567890
Salario: $1860.0

Diferencias Clave

Aspecto Clase Abstracta Clase Concreta
Instanciación ❌ No se puede crear objetos directamente ✅ Se pueden crear objetos
Métodos Puede tener métodos abstractos y concretos Solo métodos concretos (implementados)
Propósito Servir como plantilla para otras clases Representar objetos específicos del mundo real
Palabra clave abstract class class

💡 Punto clave: En Java no existen atributos abstractos. La palabra abstract solo se utiliza para métodos y clases. ¿Por qué? Porque un atributo no tiene comportamiento, solo almacena información. Lo que necesita implementación es el comportamiento (los métodos), no los datos (los atributos).

¿Cuándo Usar Clases Abstractas?

Las clases abstractas son útiles cuando tienes un grupo de clases relacionadas que comparten características comunes pero tienen comportamientos específicos diferentes. Por ejemplo, todos los empleados tienen nombre y cédula (atributos comunes), y todos deben poder calcular su salario (comportamiento común), pero cada tipo de empleado lo calcula de forma diferente (comportamiento específico).

Otros ejemplos del mundo real podrían ser una clase abstracta Vehiculo con subclases Auto, Moto y Camion, o una clase abstracta FiguraGeometrica con subclases Circulo, Rectangulo y Triangulo.
