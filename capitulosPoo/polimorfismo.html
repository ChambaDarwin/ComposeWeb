Polimorfismo en Java

Una forma, múltiples comportamientos

¿Qué es el Polimorfismo?

El polimorfismo es uno de los pilares fundamentales de la Programación Orientada a Objetos. Permite que una misma referencia de una clase padre pueda apuntar a objetos de diferentes clases hijas, y que cada objeto ejecute su propia versión de un método compartido.

Definición clave: El polimorfismo permite que una única referencia represente múltiples tipos de objetos, y que un mismo método se comporte de manera diferente según el objeto real al que apunte la referencia.

💡 Analogía del Control Remoto: Imagina que tienes un control remoto universal (la referencia). Puedes usarlo para controlar diferentes dispositivos: un televisor, un reproductor de DVD o un equipo de música. Cuando presionas el botón "Play" (el método), cada dispositivo responde de manera diferente: el TV cambia de canal, el DVD reproduce una película y el equipo reproduce música. Es el mismo botón, pero el comportamiento cambia según el dispositivo conectado.

Ejemplo Práctico

Veamos cómo funciona el polimorfismo con un ejemplo de empleados en una empresa. Todos los empleados tienen un salario, pero cada tipo de empleado lo calcula de forma diferente.

Primero, creamos una referencia de tipo Empleado (la clase padre):

Empleado e;  // Referencia de tipo Empleado

Ahora, esta referencia puede apuntar a cualquier objeto de las clases hijas y ejecutar su propia implementación del método salario():

// La referencia 'e' apunta a un objeto Docente
e = new Docente();
System.out.println(e.salario());  // Ejecuta salario() de Docente

// La misma referencia 'e' ahora apunta a un objeto Administrativo
e = new Administrativo();
System.out.println(e.salario());  // Ejecuta salario() de Administrativo

// Ahora apunta a un objeto Obrero
e = new Obrero();
System.out.println(e.salario());  // Ejecuta salario() de Obrero

Observa que: La referencia e permanece del mismo tipo (Empleado), pero los objetos a los que apunta son diferentes. Cada llamada a salario() ejecuta la implementación específica de cada clase hija.

💡 Analogía del Actor: Piensa en un actor profesional (la referencia). En una película puede interpretar a un doctor, en otra a un policía y en otra a un maestro. Es la misma persona (referencia), pero su comportamiento y forma de actuar cambia según el personaje que interprete (el objeto real). Cuando le piden "actuar" (llamar al método), lo hace según el rol que esté desempeñando en ese momento.

Ventajas del Polimorfismo

Ventaja Descripción
Flexibilidad El código puede trabajar con objetos de diferentes tipos sin conocer su tipo específico en tiempo de compilación.
Reutilización Una misma función o método puede trabajar con múltiples tipos de objetos relacionados.
Mantenibilidad Es más fácil agregar nuevas clases hijas sin modificar el código existente que usa la clase padre.
Extensibilidad El sistema puede crecer añadiendo nuevas clases que hereden de la clase base sin afectar el funcionamiento actual.

Ejemplo Completo del Sistema

// Clase abstracta padre
abstract class Empleado {
    public abstract float salario();
}

// Clases hijas con implementaciones específicas
class Docente extends Empleado {
    @Override
    public float salario() {
        return 2500.0f;  // Salario base de docente
    }
}

class Administrativo extends Empleado {
    @Override
    public float salario() {
        return 2000.0f;  // Salario base de administrativo
    }
}

class Obrero extends Empleado {
    @Override
    public float salario() {
        return 1800.0f;  // Salario base de obrero
    }
}

// Clase principal que demuestra el polimorfismo
public class Main {
    public static void main(String[] args) {
        Empleado e;  // Una sola referencia
        
        // La magia del polimorfismo
        e = new Docente();
        System.out.println("Salario Docente: $" + e.salario());
        
        e = new Administrativo();
        System.out.println("Salario Administrativo: $" + e.salario());
        
        e = new Obrero();
        System.out.println("Salario Obrero: $" + e.salario());
    }
}

💡 Analogía Final - La Toma de Corriente: Una toma de corriente (la referencia de tipo Empleado) puede recibir diferentes electrodomésticos: una licuadora, una tostadora o una cafetera (diferentes objetos de clases hijas). Cuando enciendes el interruptor (llamas al método), cada aparato hace su trabajo específico: licuar, tostar o hacer café. Es la misma toma de corriente, pero el resultado cambia según lo que esté conectado.

Conceptos Clave para Recordar

1. Una referencia, múltiples objetos: Una variable de tipo clase padre puede referenciar objetos de cualquier clase hija.

2. Comportamiento dinámico: El método que se ejecuta se determina en tiempo de ejecución según el objeto real, no según el tipo de la referencia.

3. Contrato compartido: Todas las clases hijas deben implementar los métodos definidos en la clase padre, pero cada una lo hace a su manera.

El polimorfismo es esencial porque: Permite escribir código más genérico y flexible que puede trabajar con familias enteras de objetos relacionados sin necesidad de conocer sus detalles específicos de implementación.
