🔨 Constructor

Inicialización de Objetos en Java

1¿Qué es el Constructor?

El constructor es un método especial que se utiliza para inicializar las propiedades de una clase en el momento en que se crea el objeto. Es como darle valores iniciales a las características de un objeto cuando nace.

🎯 Características del Constructor

  • Tiene el mismo nombre que la clase
  • No tiene tipo de retorno (ni siquiera void)
  • Se ejecuta automáticamente al crear un objeto con new
  • Se usa para inicializar atributos del objeto

2Cómo Crear un Constructor

Para crear un constructor seguimos esta estructura:

public Persona(String nombre, int edad) {
    this.nombre = nombre;
    this.edad = edad;
}

Análisis del código:

  • public → Modificador de acceso que permite crear objetos desde cualquier parte
  • Persona → Nombre del constructor (igual al nombre de la clase)
  • (String nombre, int edad) → Parámetros que recibe el constructor
  • this.nombre = nombre → Asigna el parámetro al atributo de la clase

💡 ¿Qué es "this"?

La palabra clave this hace referencia al objeto actual que está siendo creado o utilizado. Sirve para diferenciar entre un parámetro del constructor y un atributo de la clase cuando tienen el mismo nombre.

3Crear Objetos con Constructor

Ahora cuando creemos un nuevo objeto, debemos pasarle argumentos al constructor:

Persona persona = new Persona("Jorge", 34);
System.out.println(persona.nombre);
System.out.println(persona.edad);
Jorge
34

Los valores "Jorge" y 34 son los argumentos que se pasan al constructor y se asignan automáticamente a las propiedades del objeto.

4Crear Múltiples Objetos

Podemos crear varios objetos de la misma clase con diferentes valores. Cada objeto tendrá su propia información independiente:

Persona persona1 = new Persona("Jorge", 34);
System.out.println(persona1.nombre);
System.out.println(persona1.edad);

Persona persona2 = new Persona("Lucas", 12);
System.out.println(persona2.nombre);
System.out.println(persona2.edad);
Jorge
34
Lucas
12

Como podemos observar, tenemos dos objetos con información completamente diferente. Cada uno tiene sus propios valores en sus atributos.

🔍 Ventajas del Constructor

  • Garantiza la inicialización: Los objetos siempre tienen valores válidos desde su creación
  • Código más limpio: Evita tener que asignar cada atributo manualmente después de crear el objeto
  • Validación de datos: Puedes agregar lógica para validar los valores antes de asignarlos

5Constructor con Entrada del Usuario

Podemos combinar el constructor con la entrada de datos del usuario usando Scanner:

import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        System.out.print("Ingrese el nombre: ");
        String nombre = sc.nextLine();
        
        System.out.print("Ingrese la edad: ");
        int edad = sc.nextInt();
        
        // Crear objeto con los datos ingresados
        Persona persona = new Persona(nombre, edad);
        
        System.out.println("\n--- Datos de la persona ---");
        System.out.println("Nombre: " + persona.nombre);
        System.out.println("Edad: " + persona.edad);
    }
}

📝 Actividades Prácticas

Actividad 1: Crea una clase llamada Estudiante con tres atributos (nombre, carrera, promedio) y un constructor para inicializarlos. Luego crea un objeto donde los argumentos sean ingresados por el usuario utilizando Scanner.

Actividad 2: Si tu clase Persona tiene un atributo tipo int (edad) y otro tipo String (nombre), crea dos métodos que devuelvan estos valores:

public static String pedirNombre() {
    Scanner sc = new Scanner(System.in);
    System.out.print("Ingrese nombre: ");
    return sc.nextLine();
}

public static int pedirEdad() {
    Scanner sc = new Scanner(System.in);
    System.out.print("Ingrese edad: ");
    return sc.nextInt();
}

Luego crea un nuevo objeto utilizando estos métodos como argumentos del constructor:

Persona persona = new Persona(pedirNombre(), pedirEdad());
