Anuncios Interstitial
con Google AdMob

¿Qué es un anuncio Interstitial?

Los anuncios interstitial son anuncios de pantalla completa que cubren la interfaz de la aplicación hasta que el usuario los cierra. Están diseñados para mostrarse en pausas o transiciones naturales del flujo de la aplicación, como entre niveles de un juego o al finalizar una tarea.

Prerequisito

Antes de continuar, asegúrate de haber inicializado el SDK de Google AdMob, declarado el APP_ID en el AndroidManifest.xml, añadido el permiso INTERNET y agregado la dependencia de AdMob en build.gradle. Puedes revisar esa configuración en el capítulo anterior.

Cargar el anuncio en el ViewModel

Para gestionar anuncios que ocupan la pantalla completa utilizamos el patrón ViewModel. Dentro de él crearemos una función que solicita y carga un anuncio desde los servidores de AdMob. Para esto, la clase InterstitialAd expone el método estático load().

Kotlin · ViewModel

Parámetros de InterstitialAd.load()

El método estático load() recibe cuatro parámetros obligatorios:

PARAM 01
context: Context

Contexto de la aplicación o actividad necesario para inicializar la solicitud.

PARAM 02
adUnitId: String

Identificador de la unidad de anuncio de AdMob. En desarrollo se usa el ID de prueba de Google.

PARAM 03
adRequest: AdRequest

Objeto AdRequest que encapsula la solicitud de carga al servidor de AdMob.

PARAM 04
callback: InterstitialAdLoadCallback

Callback que notifica el resultado de la carga: éxito (onAdLoaded) o error (onAdFailedToLoad).

ID de prueba

El ID ca-app-pub-3940256099942544/1033173712 es el ID de prueba oficial de Google. Nunca uses un ID real durante el desarrollo; puedes recibir sanciones en tu cuenta de AdMob.

Estado reactivo con StateFlow

Para que la UI pueda reaccionar a la disponibilidad del anuncio, almacenamos el objeto InterstitialAd en un MutableStateFlow. Cuando el anuncio carga correctamente se asigna el objeto; cuando falla o es descartado, se establece en null.

Kotlin · Estado
Por qué StateFlow

Al usar StateFlow, cualquier composable o fragmento que observe este estado se recompondrá automáticamente cuando el anuncio esté disponible o sea descartado, sin necesidad de lógica adicional en la UI.

Mostrar el anuncio

Para presentar el interstitial en pantalla completa se requiere una referencia a la Activity activa. Dado que el ViewModel recibe un Context, se realiza un cast seguro a Activity usando el operador as?. Si la conversión falla retorna null y el bloque condicional evita un crash.

Callbacks de pantalla completa

La clase FullScreenContentCallback expone métodos para reaccionar a eventos del ciclo de vida del anuncio en pantalla completa. El más importante durante este flujo es onAdDismissedFullScreenContent().

  • 1 onAdDismissedFullScreenContent() — Se ejecuta cuando el usuario cierra el anuncio (presiona la X o el anuncio concluye). Es el momento ideal para reiniciar la carga del siguiente anuncio.
  • 2 onAdFailedToShowFullScreenContent() — Se invoca si el anuncio no pudo mostrarse correctamente. Aquí también conviene limpiar el estado y reintentarlo.
  • 3 onAdShowedFullScreenContent() — Confirma que el anuncio se mostró exitosamente. Útil para pausar música, timers o cualquier lógica de la app.
Buena práctica

Dentro de onAdDismissedFullScreenContent(), además de llamar a getInterstitial(), puedes exponer una función lambda o un SharedFlow para que la UI ejecute acciones como navegar a la siguiente pantalla o actualizar el estado del juego.

Flujo completo

El siguiente diagrama resume el ciclo de vida de un anuncio interstitial gestionado desde el ViewModel:

  • 1La app llama a getInterstitial(context) al iniciar o tras cerrar el anuncio anterior.
  • 2AdMob responde con onAdLoaded: el StateFlow se actualiza con el objeto InterstitialAd.
  • 3En el momento oportuno (transición, fin de nivel, etc.) la UI llama a showInterstitial(context).
  • 4El anuncio se presenta en pantalla completa sobre la Activity.
  • 5El usuario cierra el anuncio → onAdDismissedFullScreenContent() limpia el estado y solicita un nuevo anuncio automáticamente.
