para realizar la navegacion entre pantallas debemos comprender los siguientes conceptos navController gestiona la navegacion y backstack y me permite navegar entre pantallas y el navHost define las rutas y que composable mostrar en cada ruta para obtener una instancia de navController se utiliza rememberNavController

val navController =  rememberNavController()

una vez que tenemos una instancia de NavHostController utilizamos el NavHost() que recibe dos parametros un objeto de la clase NavHostController y un startDestination que es la ruta inicial que se va a mostrar el NavHost es un contenedor de pantallas que muestra una pantalla a la vez dentro de las llaves del navHost definimos las rutas utilizando composable

@Composable
fun contenedorNavHost(paddingValues: PaddingValues) {
    val navController = rememberNavController()
    NavHost(
        navController = navController,
        startDestination = "Home"
    ) {
        composable("Home") {
            PantallaHome(navController)
        }
        composable("Detalle") {
            PantallaDetalle(navController)
        }
    }
}

para navegar entre pantallas utilizamos el metodo navigate del objeto navController que se pasa por parametro a la funcion composable PantallaHome y PantallaDetalle y utilizamos popBackStack para regresar hacia atras


    @Composable
fun PantallaHome(navController: NavHostController) {
    Scaffold {
        Column(
            modifier = Modifier
                .padding(it)
                .background(Color.Blue)
                .fillMaxSize()
        )
        {
            Text("pantalla Home")
            Button(onClick = {
                navController.navigate("Detalle")
                
            }) {
                Text("Navegar Detalla")
            }
        }
    }
}

de esta forma podemos navegar entre pantallas utilizando botones es decir que cuando precione el boton navegamos hacia la pantalla detalle ahora lo haremos utilizando el bottomBar que es mucho mejor y mas elegante lo primero que haremos sera crear una sealed class

Que es una sealed class

define un conjunto limitado de subclases que el compilador conoce en tiempo de compilacion cuando creamos una sealed class con un constructor con parametros estamos definiendo un contrado que las subclases deben cumplir, cada subclase debe invoca explicitamente al constructor de la clase sellada pasando los parametros necesarios, una clase sellada no se puede instanciar directamente sino a traves de las subclases

sealed class Items(
    val titulo:String,
    val icon:Int,
    val route:String
){
    object Home:Items("Home", R.drawable.ic_favorite,"Home")
    object Detalle:Items("Detalle",R.drawable.ic_detalle,"Detalle")
}

Que es el backstack

el backstack es una estructura interna del NavController que guardar el historial de pantallas visitadas en la navegacion

BottomNavigation actua como un contendor que agrupa los items de navegacion en la parte inferior de la pantalla

  1. popToUp evita pantallas duplicadas eliminando el back stack
  2. saveState guarda el estado de la pantalla actual
  3. launchSingleTop evita abrir la pantalla dos veces
  4. restoreState restaura el estado anterior al volver
