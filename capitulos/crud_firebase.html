    implementation("androidx.navigation:navigation-compose:2.8.4")

🔗 Obtener una referencia a la base de datos

Cuando seleccionas Realtime Database en Firebase, obtienes una URL raíz similar a esta:

https://pruebas-a5600-default-rtdb.firebaseio.com/

Para obtener una referencia a un nodo específico de la base de datos:

val database = FirebaseDatabase.getInstance().getReference("products")
¿Qué hace getReference("products")?

Este método obtiene una referencia al nodo products de la base de datos. Esta referencia apunta a la ruta /products, y todas las operaciones (como setValue, push, etc.) se realizarán dentro de este nodo. Si el nodo products no existe, Firebase lo creará automáticamente cuando escribas datos en él.

Ejemplo de ruta completa:

https://pruebas-a5600-default-rtdb.firebaseio.com/products

➕ Insertar datos (CREATE)

Para insertar datos en la base de datos, utilizamos el método setValue(). Antes de insertar, es importante crear una referencia a un subnodo único usando push().key, ya que si no lo hacemos, los datos se sobrescribirán entre sí.

val uid = database.push().key!!
val product = Product(id = uid, precio = 34.5f, nameProduct = "teclado")
database.child(uid).setValue(product)

Explicación:

  • push().key genera un ID único para cada producto
  • child(uid) crea una referencia al subnodo con ese ID
  • setValue(product) guarda el objeto en ese nodo

📖 Leer datos (READ)

Para recuperar los registros de la base de datos, utilizamos el método addValueEventListener(). Este método recibe como parámetro un ValueEventListener, que es una interfaz con métodos que debemos implementar.

Usamos object para crear un objeto anónimo que implemente esta interfaz. Esto es útil cuando solo necesitamos usar la implementación una vez, evitando crear una clase completa.

database.addValueEventListener(object : ValueEventListener {
    override fun onDataChange(snapshot: DataSnapshot) {
        val lista = mutableListOf<Product>()
        
        for (data in snapshot.children) {
            val product = data.getValue(Product::class.java)
            product?.let {
                lista.add(it)
            }
        }
    }

    override fun onCancelled(error: DatabaseError) {
        // Manejar errores
    }
})
¿Qué hace getValue(Product::class.java)?

Este método toma el JSON almacenado en ese nodo y lo deserializa (convierte) a un objeto de tipo Product.

Proceso de deserialización:

  1. Firebase lee el JSON en ese nodo
  2. Compara las propiedades del JSON con los atributos de la clase Product
  3. Crea un objeto Product y asigna los valores del JSON a sus propiedades
⚠️ Importante:

La clase Product debe tener un constructor vacío para que Firebase pueda crear instancias del objeto correctamente.

data class Product(
    var id: String = "",
    var precio: Float = 0f,
    var nameProduct: String = ""
)

🗑️ Eliminar datos (DELETE)

Para eliminar un registro, necesitamos el ID del producto que queremos eliminar:

fun eliminarProducto(id: String) {
    database.child(id).removeValue()
}

¿Cómo funciona?

Recordemos que database apunta al nodo products:

https://pruebas-a5600-default-rtdb.firebaseio.com/products

Cuando usamos child(id), creamos una referencia a un registro específico:

https://pruebas-a5600-default-rtdb.firebaseio.com/products/-Oj_zJL9UanHWHcO_LgO

Finalmente, removeValue() elimina ese registro específico de la base de datos.

✏️ Modificar datos (UPDATE)

Para modificar un registro, seguimos el mismo concepto que para eliminar, pero en lugar de usar removeValue(), utilizamos setValue():

fun modificarProducto(id: String, productoActualizado: Product) {
    database.child(id).setValue(productoActualizado)
}

Proceso:

  1. Obtenemos la referencia al nodo específico usando child(id)
  2. Usamos setValue() con el objeto actualizado
  3. Firebase reemplaza los datos antiguos con los nuevos

Ejemplo completo:

val productoModificado = Product(
    id = "-Oj_zJL9UanHWHcO_LgO",
    precio = 45.0f,
    nameProduct = "Teclado Mecánico"
)

database.child(productoModificado.id).setValue(productoModificado)

📋 Resumen de operaciones CRUD

Operación Método Descripción
Crear child(uid).setValue(objeto) Inserta un nuevo registro con ID único
Leer addValueEventListener() Escucha cambios y recupera datos
Actualizar child(id).setValue(objetoActualizado) Modifica un registro existente
Eliminar child(id).removeValue() Elimina un registro específico
