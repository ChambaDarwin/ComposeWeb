AdMob · Jetpack Compose · Android

Banner publicitario
con AdMob

Cómo integrar un banner de Google AdMob en una aplicación Android con Jetpack Compose.

01

Configurar cuenta de AdMob

El primer paso es crear una cuenta en Google AdMob (admob.google.com), registrar la aplicación y crear una unidad de anuncio de tipo Banner. AdMob te proporcionará dos identificadores:

1
App ID

Identifica tu aplicación dentro de AdMob. Se declara en el AndroidManifest.xml.

2
Ad Unit ID

Identifica la unidad de anuncio específica (banner, intersticial, etc.). Se usa al cargar el anuncio en el código.

ℹ️
IDs de prueba para desarrollo Durante el desarrollo nunca uses tu ID real. Google proporciona IDs de prueba que muestran anuncios de ejemplo sin afectar tus métricas ni arriesgar el bloqueo de tu cuenta.
  • App ID de prueba: ca-app-pub-3940256099942544~3347511713
  • Banner Ad Unit de prueba: ca-app-pub-3940256099942544/6300978111
02

AndroidManifest.xml

Dentro de la etiqueta <application>, agrega el App ID de AdMob como metadato y declara el permiso de acceso a Internet, ya que los anuncios se cargan desde la red.

AndroidManifest.xml 
<!-- Permiso de internet -->
<uses-permission android:name="android.permission.INTERNET" />

<application>

    <!-- App ID de AdMob (usar ID real solo en producción) -->
    <meta-data
        android:name="com.google.android.gms.ads.APPLICATION_ID"
        android:value="ca-app-pub-3940256099942544~3347511713" />

</application>
⚠️
Sustituye el ID antes de publicar Reemplaza el valor de android:value por el App ID real de tu aplicación en AdMob antes de subir la app a Google Play.
03

Dependencia de AdMob

Agrega la dependencia de Google Play Services para anuncios en el archivo build.gradle.kts del módulo app:

app/build.gradle.kts 
dependencies {
    implementation("com.google.android.gms:play-services-ads:24.3.0")
}

Tras agregarla, sincroniza el proyecto con Sync Now.

04

Inicializar AdMob

Antes de cargar cualquier anuncio, es obligatorio inicializar el SDK de AdMob. Esto se hace una sola vez en el método onCreate() de la MainActivity.

MainActivity.kt 
override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)

    // Inicializa el SDK de AdMob antes de cargar cualquier anuncio
    MobileAds.initialize(this)

    setContent {
        // ...
    }
}
✅
¿Por qué inicializar primero? Si intentas cargar un anuncio antes de inicializar MobileAds, el SDK no estará preparado y el banner no se mostrará. La inicialización es asíncrona y solo necesita ejecutarse una vez en toda la app.
05

Crear el Banner

AdView es la vista nativa de Android SDK que muestra el banner, pero no es un Composable. Para usar vistas tradicionales de Android dentro de Jetpack Compose se utiliza AndroidView, que actúa como puente entre ambos sistemas.

AndroidView es un Composable especial que envuelve cualquier vista clásica de Android (View) y la integra dentro del árbol de composición de Jetpack Compose. Recibe un parámetro factory, una lambda que crea y configura la vista, y que expone el contexto necesario para instanciarla.
BannerAdView.kt 
@Composable
fun BannerAd() {
    AndroidView(
        factory = { context ->
            AdView(context).apply {
                setAdSize(AdSize.BANNER)
                adUnitId = "ca-app-pub-3940256099942544/6300978111" // ID de prueba
                loadAd(AdRequest.Builder().build())
            }
        }
    )
}

Métodos explicados

setAdSize() Define el tamaño del banner. AdSize.BANNER es el tamaño estándar de 320×50 dp. Google ofrece otros tamaños como LARGE_BANNER o MEDIUM_RECTANGLE.
adUnitId El identificador único de la unidad de anuncio. Durante desarrollo se usa el ID de prueba; en producción se reemplaza por el ID real obtenido en la consola de AdMob.
AdRequest.Builder() Crea un objeto que representa la solicitud de carga del anuncio. Se pueden añadir configuraciones adicionales antes de llamar a build().
loadAd() Envía la solicitud a los servidores de Google y descarga el anuncio para mostrarlo en el AdView.

Flujo de carga del banner

AdView creado
→
setAdSize()
→
adUnitId asignado
→
AdRequest.Builder().build()
→
loadAd()
→
Banner visible
💡
Uso en la pantalla Puedes colocar BannerAd() como cualquier otro Composable, por ejemplo al final de un Column o dentro de un Scaffold en la parte inferior de la pantalla.
