✅ Asserts de JUnit
Método Descripción Ejemplo
JUnit
assertEquals(expected, actual)
 Verifica que dos valores sean iguales. Es el assert más común. assertEquals(10, resultado)
JUnit
assertNotEquals(unexpected, actual)
 Verifica que dos valores no sean iguales. assertNotEquals(0, lista.size)
JUnit
assertTrue(condition)
 Verifica que una condición sea verdadera. assertTrue(usuario.isActivo)
JUnit
assertFalse(condition)
 Verifica que una condición sea falsa. assertFalse(lista.isEmpty())
JUnit
assertNull(value)
 Verifica que el valor sea null. assertNull(usuario.nombre)
JUnit
assertNotNull(value)
 Verifica que el valor no sea null. assertNotNull(viewModel)
JUnit
assertSame(expected, actual)
 Verifica que ambos objetos sean la misma instancia (misma referencia en memoria). assertSame(obj1, obj2)
JUnit
assertNotSame(unexpected, actual)
 Verifica que no sean la misma instancia. assertNotSame(obj1, obj2)
JUnit
assertArrayEquals(expectedArray, actualArray)
 Compara dos arrays elemento por elemento. assertArrayEquals(arrayOf(1,2), resultado)
JUnit
assertIterableEquals(expected, actual)
 Compara dos colecciones/listas elemento por elemento. assertIterableEquals(listOf(1,2,3), lista)
JUnit
assertLinesMatch(expectedLines, actualLines)
 Compara listas de strings (útil para texto multilínea, soporta expresiones regulares). assertLinesMatch(expected, actual)
JUnit
assertThrows<T: Throwable> { ... }
 Verifica que se lance una excepción específica dentro del bloque. assertThrows<IllegalArgumentException> { funcion() }
JUnit
assertTimeout(duration) { ... }
 Verifica que un bloque termine dentro de un tiempo máximo. assertTimeout(Duration.ofMillis(200)) { tarea() }
JUnit
assertDoesNotThrow { ... }
 Verifica que el bloque no lance ninguna excepción. assertDoesNotThrow { funcionSegura() }
JUnit
assertEquals(expected, actual, delta)
 Compara números con tolerancia (útil para floats/doubles). assertEquals(3.14, resultado, 0.001)
MockK
every { ... } returns ...
 Define el comportamiento del mock (stubbing). Cuando se llama al método del mock, retorna el valor especificado. every { repo.getUser("1") } returns user
MockK
verify { ... }
 Verifica que una llamada al mock ocurrió al menos una vez. verify { repo.save(usuario) }
MockK
verify(exactly = n) { ... }
 Verifica que la llamada ocurrió exactamente n veces. verify(exactly = 1) { repo.insert(any()) }
MockK
verifyOrder { ... }
 Verifica que las llamadas ocurrieron en el orden indicado (pueden existir otras llamadas entre medio). verifyOrder { a(); b(); c() }
MockK
verifySequence { ... }
 Verifica que exactamente esas llamadas ocurrieron en ese orden, sin llamadas intermedias no especificadas. verifySequence { abrir(); guardar(); cerrar() }
MockK Coroutine
coEvery { ... } returns ...
 Stubbing para funciones suspend (coroutines). Versión de every para código asíncrono. coEvery { repo.fetch() } returns data
MockK Coroutine
coVerify { ... }
 Verificación para funciones suspend. Versión de verify para coroutines. coVerify { repo.fetch() }
MockK
slot(), capture(...)
 Captura argumentos pasados a la llamada para realizar aserciones posteriores sobre ellos. val slot = slot<String>()
verify { repo.save(capture(slot)) }
assertEquals("hola", slot.captured)
MockK
confirmVerified(mock)
 Confirma que no quedan interacciones sin verificar con el mock. Útil para asegurar que no hubo llamadas inesperadas. confirmVerified(repo)
MockK
clearMocks(...)
 Limpia el historial e interacciones de los mocks. Útil para resetear mocks entre tests. clearMocks(repo)
