Autenticación con Firebase: Email/Password

Conceptos Fundamentales

Lambda: Una expresión lambda es una función anónima que se escribe de forma concisa. En Kotlin, se representa entre llaves { }.
Callback: Es una función que pasas como parámetro a otra función para que sea ejecutada posteriormente. No es una sintaxis específica, sino un patrón de diseño.

Ejemplo de Lambda y Callback

// Lambda - función anónima
val saludo = { println("Hola") }

// Callback - función pasada como parámetro
fun ejecutar(callback: () -> Unit) {
    println("Antes")
    callback() // Se ejecuta el callback
    println("Después")
}

// Uso
ejecutar { println("Hola desde el callback") }

Configuración Inicial

Para implementar la autenticación con email y contraseña en Firebase, necesitas agregar las siguientes dependencias a tu archivo build.gradle:

// Firebase BOM (Bill of Materials) - gestiona versiones automáticamente
implementation(platform("com.google.firebase:firebase-bom:33.7.0"))

// Firebase Authentication
implementation("com.google.firebase:firebase-auth-ktx")

Crear la Instancia de FirebaseAuth

private val firebaseAuth = FirebaseAuth.getInstance()

¿Qué es FirebaseAuth?

FirebaseAuth es la clase principal que gestiona toda la autenticación de usuarios en tu aplicación. Proporciona múltiples funcionalidades integradas:

  • Persistencia automática: La sesión del usuario se mantiene incluso después de cerrar la aplicación
  • Seguridad: Las contraseñas se cifran automáticamente
  • Verificación de email: Permite enviar correos de verificación
  • Recuperación de contraseña: Utilizando sendPasswordResetEmail()
  • Plan gratuito: Hasta 50,000 autenticaciones mensuales sin costo

Métodos Principales de FirebaseAuth

Método Descripción
createUserWithEmailAndPassword(email, password) Crea un nuevo usuario con email y contraseña
signInWithEmailAndPassword(email, password) Inicia sesión con credenciales existentes
signOut() Cierra la sesión del usuario actual
currentUser Obtiene el usuario actual (retorna null si no hay sesión activa)

Listeners de Firebase

1. addOnCompleteListener

Se ejecuta siempre, sin importar si la operación fue exitosa o falló. Es útil cuando necesitas manejar ambos casos (éxito y error) en un mismo lugar.

firebaseAuth.createUserWithEmailAndPassword(email, password)
    .addOnCompleteListener { task ->
        if (task.isSuccessful) {
            // Operación exitosa
            val authResult = task.result
            val user = authResult?.user
            println("Usuario creado: ${user?.email}")
        } else {
            // Error en la operación
            val error = task.exception
            println("Error: ${error?.message}")
        }
    }
Objeto Task<AuthResult>:
• isSuccessful: Boolean - Indica si la operación fue exitosa
• exception: Exception? - Contiene el error si hubo uno
• result: AuthResult? - Contiene el resultado si fue exitoso

2. addOnSuccessListener

Se ejecuta solo si la operación fue exitosa. Recibe directamente el objeto AuthResult, lo que simplifica el código cuando solo te interesa el caso de éxito.

firebaseAuth.createUserWithEmailAndPassword(email, password)
    .addOnSuccessListener { authResult ->
        // Solo se ejecuta si fue exitoso
        val user = authResult.user
        println("Usuario: ${user?.email}")
    }
    .addOnFailureListener { exception ->
        // Solo se ejecuta si hubo error
        println("Error: ${exception.message}")
    }

Objeto FirebaseUser

Cuando la autenticación es exitosa, obtienes un objeto FirebaseUser que contiene la información del usuario autenticado:

Propiedad Tipo Descripción
uid String ID único del usuario
email String? Email del usuario
displayName String? Nombre del usuario
photoUrl Uri? URL de la foto de perfil

Implementación en ViewModel

Para exponer el estado de la autenticación a la UI, utilizamos StateFlow, que es un flujo de datos observable en Kotlin:

class AuthViewModel : ViewModel() {
    private val firebaseAuth = FirebaseAuth.getInstance()
    
    // StateFlow privado (mutable)
    private val _firebaseUser = MutableStateFlow<FirebaseUser?>(null)
    
    // StateFlow público (inmutable)
    val firebaseUser: StateFlow<FirebaseUser?> = _firebaseUser
    
    fun createUser(email: String, password: String) {
        firebaseAuth.createUserWithEmailAndPassword(email, password)
            .addOnCompleteListener { task ->
                if (task.isSuccessful) {
                    // Actualizar el estado con el usuario autenticado
                    _firebaseUser.value = task.result?.user
                } else {
                    // Actualizar con null en caso de error
                    _firebaseUser.value = null
                }
            }
    }
}

Integración con Jetpack Compose

En la UI, utilizamos collectAsState() para convertir el StateFlow en un State observable por Compose:

@Composable
fun RegisterScreen(viewModel: AuthViewModel) {
    val context = LocalContext.current
    
    // Observar el estado del usuario
    val firebaseUser by viewModel.firebaseUser.collectAsState()
    
    Column {
        // UI de registro...
        
        Button(onClick = {
            viewModel.createUser(email, password)
        }) {
            Text("Registrar")
        }
        
        // Reaccionar al cambio de estado
        LaunchedEffect(firebaseUser) {
            firebaseUser?.let { user ->
                println("Email: ${user.email}")
                println("Name: ${user.displayName}")
                println("UID: ${user.uid}")
                Toast.makeText(
                    context, 
                    "Usuario creado exitosamente", 
                    Toast.LENGTH_SHORT
                ).show()
            }
        }
    }
}
Nota importante: Al crear un usuario con email y contraseña, displayName será null por defecto, ya que solo se proporciona el email y contraseña. Para establecer un nombre, debes actualizarlo explícitamente.

val user = FirebaseAuth.getInstance().currentUser

val profileUpdates = userProfileChangeRequest {

user?.updateProfile(profileUpdates)
    ?.addOnCompleteListener { task ->

        }
    }
Recomendación: Para almacenar información adicional del usuario (edad, dirección, preferencias, etc.), es mejor utilizar Firestore o Realtime Database:
  • Auth: Solo para autenticación
  • Firestore/Realtime DB: Para datos del perfil y otra información

Manejo de Errores con Sealed Classes

Una sealed class representa un conjunto cerrado de tipos. Solo las clases definidas dentro del mismo archivo pueden heredar de ella, lo que garantiza seguridad en tiempo de compilación.

Características de Sealed Classes

  • Define un número limitado de subtipos posibles
  • Todas las subclases deben estar en el mismo archivo
  • No se puede instanciar directamente
  • Permite usar when exhaustivo sin else
  • Implementa polimorfismo de manera segura

Implementación de UiState

sealed class UiState<out T> {
    // Estado inicial
    object Idle : UiState<Nothing>()
    
    // Estado de carga
    object Loading : UiState<Nothing>()
    
    // Estado de error con mensaje
    data class Error(val message: String) : UiState<Nothing>()
    
    // Estado de éxito con datos
    data class Success<T>(val data: T?) : UiState<T>()
}
Genéricos en Sealed Classes:
• Nothing: Tipo especial que representa la ausencia de valor
• T: Tipo genérico que se define al momento de usar la clase
• out T: Covarianza - el tipo T solo puede ser retornado, no recibido como parámetro

Uso en ViewModel

class AuthViewModel : ViewModel() {
    private val firebaseAuth = FirebaseAuth.getInstance()
    
    private val _authState = MutableStateFlow<UiState<FirebaseUser>>(UiState.Idle)
    val authState: StateFlow<UiState<FirebaseUser>> = _authState
    
    fun createUser(email: String, password: String) {
        // Cambiar a estado de carga
        _authState.value = UiState.Loading
        
        firebaseAuth.createUserWithEmailAndPassword(email, password)
            .addOnSuccessListener { authResult ->
                // Estado de éxito
                _authState.value = UiState.Success(authResult.user)
            }
            .addOnFailureListener { exception ->
                // Estado de error
                _authState.value = UiState.Error(
                    exception.message ?: "Error desconocido"
                )
            }
    }
}

Manejo en la UI

@Composable
fun RegisterScreen(viewModel: AuthViewModel) {
    val context = LocalContext.current
    val authState by viewModel.authState.collectAsState()
    
    Column {
        // UI de registro...
        
        Button(onClick = {
            viewModel.createUser(email, password)
        }) {
            Text("Registrar")
        }
        
        // Manejo exhaustivo de estados
        when (val state = authState) {
            is UiState.Idle -> {
                // Estado inicial, no hacer nada
            }
            
            is UiState.Loading -> {
                CircularProgressIndicator()
            }
            
            is UiState.Error -> {
                Toast.makeText(
                    context, 
                    state.message, 
                    Toast.LENGTH_SHORT
                ).show()
            }
            
            is UiState.Success -> {
                val user = state.data
                Toast.makeText(
                    context, 
                    "Bienvenido ${user?.email}", 
                    Toast.LENGTH_SHORT
                ).show()
                // Navegar a la siguiente pantalla
            }
        }
    }
}

Errores Comunes de Firebase Auth

Código de Error Descripción
ERROR_INVALID_EMAIL El email no tiene un formato válido
ERROR_WEAK_PASSWORD La contraseña es demasiado débil (menos de 6 caracteres)
ERROR_EMAIL_ALREADY_IN_USE El email ya está registrado
ERROR_USER_NOT_FOUND No existe un usuario con ese email
ERROR_WRONG_PASSWORD La contraseña es incorrecta

Resumen de Buenas Prácticas

  1. Usar StateFlow para exponer el estado de autenticación al UI
  2. Implementar Sealed Classes para manejar diferentes estados (Loading, Success, Error)
  3. Separar responsabilidades: Auth para autenticación, Firestore/Realtime DB para datos
  4. Validar entradas antes de enviarlas a Firebase
  5. Manejar errores apropiadamente con mensajes claros para el usuario
  6. Usar LaunchedEffect en Compose para reaccionar a cambios de estado
  7. No almacenar contraseñas en texto plano ni en SharedPreferences
