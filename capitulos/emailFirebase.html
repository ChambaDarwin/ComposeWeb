🎯 Conceptos Fundamentales

Lambda y Callback

Antes de comenzar con Firebase Authentication, es importante comprender dos conceptos clave:

Lambda: Es una función anónima escrita de forma concisa. En Kotlin, se representa con la sintaxis { }.

Callback: Es una función que se pasa como parámetro a otra función para que sea ejecutada en un momento posterior, típicamente cuando se completa una operación asíncrona.

// Ejemplo de lambda
val saludar = { println("¡Hola, Firebase!") }

// Ejemplo de callback
fun ejecutarOperacion(callback: () -> Unit) {
    println("Iniciando operación...")
    // Operación asíncrona aquí
    callback() // Se ejecuta después
    println("Operación completada")
}

// Uso
ejecutarOperacion {
    println("Este es el callback ejecutándose")
}

⚙️ Configuración Inicial

Dependencias requeridas

Para utilizar Firebase Authentication, primero debemos agregar las siguientes dependencias en el archivo build.gradle del módulo:

// Firebase BOM (Bill of Materials) - gestiona las versiones automáticamente
implementation(platform("com.google.firebase:firebase-bom:33.7.0"))

// Firebase Authentication
implementation("com.google.firebase:firebase-auth-ktx")

Instancia de FirebaseAuth

Una vez agregadas las dependencias, creamos una instancia de FirebaseAuth en nuestro ViewModel o repositorio:

private val firebaseAuth = FirebaseAuth.getInstance()

¿Qué es FirebaseAuth? Es la clase principal de Firebase Authentication que gestiona toda la autenticación de usuarios en tu aplicación.

Características principales

🔐 Persistencia Automática La sesión se mantiene incluso si cierras la aplicación
🛡️ Seguridad Las contraseñas se encriptan automáticamente
✉️ Verificación de Email Envío de correos de verificación integrado
🔄 Recuperación de Contraseña Con el método sendPasswordResetEmail()
💰 Gratuito Hasta 50,000 autenticaciones mensuales sin costo
🌐 Multi-plataforma Compatible con Android, iOS, Web y más

🔧 Métodos Principales de FirebaseAuth

Método Descripción Uso
createUserWithEmailAndPassword() Crea un nuevo usuario Registro
signInWithEmailAndPassword() Inicia sesión con credenciales Login
signOut() Cierra la sesión actual Logout
currentUser Obtiene el usuario actual (null si no hay sesión) Estado de sesión
sendPasswordResetEmail() Envía email de recuperación Recuperar contraseña

👂 Listeners y Manejo de Resultados

addOnCompleteListener

Se ejecuta siempre, independientemente del resultado de la operación. Es útil cuando necesitas manejar tanto el éxito como el error en un mismo lugar.

firebaseAuth.createUserWithEmailAndPassword(email, password)
    .addOnCompleteListener { task ->
        if (task.isSuccessful) {
            // ✅ Operación exitosa
            val authResult = task.result
            val user = authResult?.user
            println("Usuario creado: ${user?.email}")
        } else {
            // ❌ Ocurrió un error
            val exception = task.exception
            println("Error: ${exception?.message}")
        }
    }

El objeto Task: Es un contenedor que encapsula el resultado de la operación asíncrona. Contiene:
• isSuccessful: Boolean
• exception: Exception?
• result: AuthResult? (solo si fue exitoso)

addOnSuccessListener & addOnFailureListener

Estos métodos separan el manejo de éxito y error en listeners independientes, proporcionando un código más limpio y organizado.

firebaseAuth.createUserWithEmailAndPassword(email, password)
    .addOnSuccessListener { authResult ->
        // ✅ Solo se ejecuta si la operación fue exitosa
        val user = authResult.user
        println("Bienvenido: ${user?.email}")
    }
    .addOnFailureListener { exception ->
        // ❌ Solo se ejecuta si ocurrió un error
        println("Error en autenticación: ${exception.message}")
    }

Nota importante: Con addOnSuccessListener, recibes directamente el objeto AuthResult porque el listener solo se ejecuta cuando la operación es exitosa.

Objeto AuthResult y FirebaseUser

El objeto AuthResult contiene información sobre el resultado de la autenticación. Su propiedad user devuelve un objeto FirebaseUser con los siguientes datos:

uid Identificador único del usuario (String)
email Correo electrónico del usuario (String?)
displayName Nombre para mostrar (String?)
photoUrl URL de la foto de perfil (Uri?)
isEmailVerified Estado de verificación del email (Boolean)
phoneNumber Número de teléfono (String?)

🔄 Integración con StateFlow y Compose

Gestión del estado en el ViewModel

Para comunicar el estado de autenticación desde el ViewModel hacia la UI, utilizamos StateFlow:

class AuthViewModel : ViewModel() {
    private val firebaseAuth = FirebaseAuth.getInstance()
    
    // StateFlow privado (mutable)
    private val _firebaseUser = MutableStateFlow(null)
    
    // StateFlow público (inmutable)
    val firebaseUser: StateFlow = _firebaseUser
    
    fun createUser(email: String, password: String) {
        firebaseAuth.createUserWithEmailAndPassword(email, password)
            .addOnSuccessListener { authResult ->
                // Actualizar el estado
                _firebaseUser.value = authResult.user
            }
            .addOnFailureListener { exception ->
                // Manejar error (veremos más adelante con Sealed Classes)
                println("Error: ${exception.message}")
            }
    }
}

Observando el estado en Compose

En tu composable, observa el StateFlow usando collectAsState():

@Composable
fun LoginScreen(viewModel: AuthViewModel) {
    // Convertir StateFlow a State observable por Compose
    val firebaseUser by viewModel.firebaseUser.collectAsState()
    val context = LocalContext.current
    
    Column(
        modifier = Modifier.fillMaxSize().padding(16.dp)
    ) {
        // ... campos de email y password
        
        Button(onClick = { 
            viewModel.createUser(email, password) 
        }) {
            Text("Registrar")
        }
        
        // Reaccionar a cambios de estado
        if (firebaseUser != null) {
            LaunchedEffect(firebaseUser) {
                Toast.makeText(
                    context,
                    "Usuario creado: ${firebaseUser?.email}",
                    Toast.LENGTH_SHORT
                ).show()
            }
        }
    }
}

collectAsState(): Convierte un Flow en un State que Compose puede observar, lo que permite que la UI se recomponga automáticamente cuando el valor cambia.

Actualizando el perfil del usuario

Por defecto, el displayName es null al crear un usuario con email/password. Para actualizarlo:

val user = FirebaseAuth.getInstance().currentUser

val profileUpdates = userProfileChangeRequest {
    displayName = "Juan Pérez"
    photoUri = Uri.parse("https://ejemplo.com/foto.jpg")
}

user?.updateProfile(profileUpdates)
    ?.addOnSuccessListener {
        println("Perfil actualizado correctamente")
    }
    ?.addOnFailureListener { exception ->
        println("Error al actualizar perfil: ${exception.message}")
    }

Recomendación: Para almacenar información adicional del usuario (biografía, preferencias, etc.), es mejor utilizar Firestore o Realtime Database. Firebase Auth solo debe gestionar la autenticación, mientras que las bases de datos manejan los datos del perfil.

🔒 Sealed Classes para Gestión de Estados

¿Qué es una Sealed Class?

Una sealed class representa un conjunto cerrado de tipos. "Cerrado" significa que solo ciertas clases pueden heredar de ella y deben estar definidas en el mismo archivo.

Ventajas:
• Representa estados mutuamente excluyentes de forma segura
• El compilador conoce todos los subtipos posibles
• Permite exhaustividad en expresiones when
• Evita errores por estados no manejados

Diseño de UiState

Creamos una sealed class genérica para representar diferentes estados de la UI:

sealed class UiState {
    // Estado inicial - no se ha realizado ninguna acción
    object Idle : UiState()
    
    // Estado de carga - operación en progreso
    object Loading : UiState()
    
    // Estado de éxito - operación completada con datos
    data class Success(val data: T?) : UiState()
    
    // Estado de error - operación falló
    data class Error(val message: String) : UiState()
}

Sobre los genéricos:
• T: Tipo genérico que se define al usar la clase
• Nothing: Tipo especial que indica que no hay valor (tipo "vacío")
• out T: Covarianza - permite devolver subtipos de T

Implementación en el ViewModel

class AuthViewModel : ViewModel() {
    private val firebaseAuth = FirebaseAuth.getInstance()
    
    private val _authState = MutableStateFlow>(UiState.Idle)
    val authState: StateFlow> = _authState
    
    fun createUser(email: String, password: String) {
        // Cambiar a estado de carga
        _authState.value = UiState.Loading
        
        firebaseAuth.createUserWithEmailAndPassword(email, password)
            .addOnSuccessListener { authResult ->
                // Éxito - actualizar con el usuario
                _authState.value = UiState.Success(authResult.user)
            }
            .addOnFailureListener { exception ->
                // Error - actualizar con mensaje de error
                _authState.value = UiState.Error(
                    exception.message ?: "Error desconocido"
                )
            }
    }
    
    fun signIn(email: String, password: String) {
        _authState.value = UiState.Loading
        
        firebaseAuth.signInWithEmailAndPassword(email, password)
            .addOnSuccessListener { authResult ->
                _authState.value = UiState.Success(authResult.user)
            }
            .addOnFailureListener { exception ->
                _authState.value = UiState.Error(
                    exception.message ?: "Error al iniciar sesión"
                )
            }
    }
}

Uso en Compose con when

En tu composable, maneja todos los estados posibles con una expresión when:

@Composable
fun AuthScreen(viewModel: AuthViewModel) {
    val authState by viewModel.authState.collectAsState()
    val context = LocalContext.current
    
    Column(
        modifier = Modifier.fillMaxSize().padding(16.dp),
        horizontalAlignment = Alignment.CenterHorizontally,
        verticalArrangement = Arrangement.Center
    ) {
        // ... campos de formulario
        
        Button(onClick = { 
            viewModel.createUser(email, password) 
        }) {
            Text("Registrar")
        }
        
        Spacer(modifier = Modifier.height(16.dp))
        
        // Manejar estados con when
        when (val state = authState) {
            is UiState.Idle -> {
                // Estado inicial - no mostrar nada especial
            }
            
            is UiState.Loading -> {
                CircularProgressIndicator(
                    modifier = Modifier.size(48.dp),
                    color = MaterialTheme.colorScheme.primary
                )
                Text("Procesando...", style = MaterialTheme.typography.bodyMedium)
            }
            
            is UiState.Success -> {
                LaunchedEffect(state) {
                    Toast.makeText(
                        context,
                        "Bienvenido ${state.data?.email}",
                        Toast.LENGTH_SHORT
                    ).show()
                    // Navegar a la siguiente pantalla
                }
            }
            
            is UiState.Error -> {
                Text(
                    text = "❌ ${state.message}",
                    color = MaterialTheme.colorScheme.error,
                    style = MaterialTheme.typography.bodyMedium
                )
                
                LaunchedEffect(state) {
                    Toast.makeText(
                        context,
                        state.message,
                        Toast.LENGTH_LONG
                    ).show()
                }
            }
        }
    }
}

Ventaja de usar when: El compilador de Kotlin verifica que manejes todos los casos posibles de la sealed class. Si agregas un nuevo estado, el compilador te alertará sobre los lugares donde falta manejarlo.

✨ Mejores Prácticas

1. Validación de datos

fun validateEmail(email: String): Boolean {
    return android.util.Patterns.EMAIL_ADDRESS.matcher(email).matches()
}

fun validatePassword(password: String): Boolean {
    return password.length >= 6
}

// En el ViewModel
fun createUser(email: String, password: String) {
    when {
        !validateEmail(email) -> {
            _authState.value = UiState.Error("Email inválido")
            return
        }
        !validatePassword(password) -> {
            _authState.value = UiState.Error("La contraseña debe tener al menos 6 caracteres")
            return
        }
    }
    
    // Continuar con el registro
    _authState.value = UiState.Loading
    // ...
}

2. Manejo de errores específicos

import com.google.firebase.auth.FirebaseAuthException

fun handleAuthException(exception: Exception): String {
    return when (exception) {
        is FirebaseAuthException -> {
            when (exception.errorCode) {
                "ERROR_EMAIL_ALREADY_IN_USE" -> 
                    "Este correo ya está registrado"
                "ERROR_INVALID_EMAIL" -> 
                    "Formato de correo inválido"
                "ERROR_WEAK_PASSWORD" -> 
                    "La contraseña es muy débil"
                "ERROR_WRONG_PASSWORD" -> 
                    "Contraseña incorrecta"
                "ERROR_USER_NOT_FOUND" -> 
                    "Usuario no encontrado"
                else -> 
                    exception.message ?: "Error de autenticación"
            }
        }
        else -> exception.message ?: "Error desconocido"
    }
}

// Uso en el listener
.addOnFailureListener { exception ->
    val errorMessage = handleAuthException(exception)
    _authState.value = UiState.Error(errorMessage)
}

3. Verificación de sesión al iniciar

class AuthViewModel : ViewModel() {
    private val firebaseAuth = FirebaseAuth.getInstance()
    
    init {
        checkCurrentUser()
    }
    
    private fun checkCurrentUser() {
        val currentUser = firebaseAuth.currentUser
        _authState.value = if (currentUser != null) {
            UiState.Success(currentUser)
        } else {
            UiState.Idle
        }
    }
}

4. Separación de responsabilidades

Es recomendable crear un repositorio para encapsular la lógica de Firebase:

// AuthRepository.kt
class AuthRepository {
    private val firebaseAuth = FirebaseAuth.getInstance()
    
    suspend fun createUser(email: String, password: String): Result {
        return try {
            val result = firebaseAuth
                .createUserWithEmailAndPassword(email, password)
                .await()
            Result.success(result.user!!)
        } catch (e: Exception) {
            Result.failure(e)
        }
    }
    
    suspend fun signIn(email: String, password: String): Result {
        return try {
            val result = firebaseAuth
                .signInWithEmailAndPassword(email, password)
                .await()
            Result.success(result.user!!)
        } catch (e: Exception) {
            Result.failure(e)
        }
    }
    
    fun signOut() {
        firebaseAuth.signOut()
    }
    
    fun getCurrentUser(): FirebaseUser? = firebaseAuth.currentUser
}

// ViewModel actualizado
class AuthViewModel(
    private val authRepository: AuthRepository
) : ViewModel() {
    
    private val _authState = MutableStateFlow>(UiState.Idle)
    val authState: StateFlow> = _authState
    
    fun createUser(email: String, password: String) {
        viewModelScope.launch {
            _authState.value = UiState.Loading
            
            authRepository.createUser(email, password)
                .onSuccess { user ->
                    _authState.value = UiState.Success(user)
                }
                .onFailure { exception ->
                    _authState.value = UiState.Error(
                        handleAuthException(exception)
                    )
                }
        }
    }
}

5. Almacenamiento de datos adicionales

// UserProfile.kt
data class UserProfile(
    val uid: String,
    val email: String,
    val displayName: String,
    val photoUrl: String? = null,
    val createdAt: Long = System.currentTimeMillis()
)

// Después de crear el usuario
firebaseAuth.createUserWithEmailAndPassword(email, password)
    .addOnSuccessListener { authResult ->
        val user = authResult.user!!
        
        // Guardar perfil en Firestore
        val userProfile = UserProfile(
            uid = user.uid,
            email = user.email!!,
            displayName = displayName
        )
        
        FirebaseFirestore.getInstance()
            .collection("users")
            .document(user.uid)
            .set(userProfile)
            .addOnSuccessListener {
                _authState.value = UiState.Success(user)
            }
    }

🎓 Conclusión

Firebase Authentication proporciona una solución robusta y fácil de implementar para la autenticación de usuarios. Al combinarla con arquitecturas modernas como MVVM, StateFlow y Sealed Classes, puedes crear aplicaciones Android seguras y mantenibles.

Recuerda siempre validar los datos del usuario, manejar errores de forma específica y separar las responsabilidades usando el patrón Repository para facilitar el testing y el mantenimiento de tu código.
