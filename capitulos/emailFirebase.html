para poder realizar la autenticacion por medio de email/password lo primero que demos realizar es obtener una instancia de Firebase para ello debemos utilizar las siguiente dependencias


    // Firebase BOM (Bill of Materials) - maneja las versiones automáticamente
    implementation(platform("com.google.firebase:firebase-bom:33.7.0"))
    
    // Firebase Authentication
    implementation("com.google.firebase:firebase-auth-ktx")

de esta creamos una instancia de FirebaseAuth 


         private val firebaseAuth = FirebaseAuth.getInstance()

FirebaseAuth es la clase principal de Firebase Authentication que maneja toda la autenticación de usuarios en tu app. tiene varios metodos que se utilizan para gestionar usuarios como por ejemplo ademas posee Persistencia automática - La sesión se mantiene aunque cierres la app Seguridad - Las contraseñas se encriptan automáticamente Verificación de email - Puedes enviar emails de verificación Recuperación de contraseña - Con sendPasswordResetEmail() Gratuito - Hasta 50,000 autenticaciones mensuales gratis

  1. createUserWithEmailAndPassword(email, password) para crear un uevo usuario
  2. signInWithEmailAndPassword(email, password) para iniciar sesion
  3. signOut() para cerrar sesion
  4. auth.currentUser conocer el usuario actual devuelve null si no hay un usuario activo

Metodos importantes

addOnCompleteListener Se ejecuta SIEMPRE, sin importar si la operación fue exitosa o falló. Uso: Cuando necesitas manejar AMBOS casos (éxito y error) en un solo lugar. tambien vas a poder observar it:Task<AuthResult> Task es un "envoltorio" que contiene: // - isSuccessful: Boolean // - exception: Exception? // - result: AuthResult? (solo si fue exitoso) cuando utilizamos el metodo result nos devuelve un objeto de la clase AuthResult y si utilizamos el metodo user nos devuelve un objeto de tipo FirebaseUser este objeto contiene la informacion del usuario autenticado como:

  1. uid es el id unico del usuario autenticado
  2. email email del usuario
  3. displayName nombre del usuario
  4. photoUrl Uri? -Url de la foto de perfil

2. addOnSuccessListener Se ejecuta SOLO si la operación fue exitosa. cuando utilizas este metodo ya te devuelve el objeto AuthResult directamente porque recuerda que solo va a ingresar si la operación es exitosa y a partir del objeto ya puedes utilizar el metodo user para obtener un objeto de la clase Firebase user y utilizar los mismos metodos que se describio anteriormente 


     firebaseAuth.createUserWithEmailAndPassword(email, password).addOnCompleteListener {
            if (it.isSuccessful) {
                val user = it.result
            } else {
                val error = it.exception
            }
        }
        firebaseAuth.createUserWithEmailAndPassword(email, password).addOnSuccessListener {

        }.addOnFailureListener {

        }

     private val _firebaseUser = MutableStateFlow(null)
    val firebaseUser: StateFlow get() = _firebaseUser

    _firebaseUser.value=user_firebaseUser.value=user
