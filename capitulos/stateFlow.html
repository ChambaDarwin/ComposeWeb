🌊 StateFlow en Kotlin

Manejo de estados reactivos de forma simple y eficiente

¿Qué es MutableStateFlow y StateFlow?

StateFlow es un contenedor de estado observable que emite actualizaciones a sus observadores cada vez que el valor cambia. Piensa en él como un canal de noticias que siempre tiene una noticia actual y notifica a sus suscriptores cuando hay algo nuevo.

🔒 StateFlow

Solo lectura. Los observadores pueden leer el estado pero no modificarlo.

✏️ MutableStateFlow

Permite leer y modificar el estado. Es privado dentro de tu ViewModel.

📺 Analogía: El Noticiero

MutableStateFlow es como el equipo de producción del noticiero que puede escribir y cambiar las noticias.

StateFlow es como la pantalla de TV que solo muestra las noticias a los espectadores, ellos no pueden modificar el contenido.

El público (UI) solo ve lo que se transmite (StateFlow), mientras que los productores (ViewModel) tienen el control editorial completo (MutableStateFlow).

Ejemplo de implementación:

private var _authentication = MutableStateFlow(AuthenticationEmail())
val authentication: StateFlow<AuthenticationEmail> get() = _authentication

⚠️ Regla de Oro: Emisión por Referencia

🔑 Concepto Clave: StateFlow solo emite cambios cuando detecta que la referencia del objeto ha cambiado, no cuando cambian sus propiedades internas.

📦 Analogía: La Caja de Entrega

Imagina que StateFlow es un servicio de entrega que solo se activa cuando entregas una caja nueva (nueva referencia).

❌ No funciona: Abrir la caja actual y cambiar lo que hay dentro. El servicio no detecta cambios porque sigue siendo la misma caja.

✅ Funciona: Crear una caja completamente nueva con el contenido actualizado. El servicio detecta la nueva caja y la entrega.

La Función copy() - Tu Mejor Aliada

En Kotlin, las data classes tienen una función especial llamada copy() que crea un nuevo objeto con las mismas propiedades, pero te permite modificar las que necesites.

Ejemplo visual:

data class User(
    val name: String,
    val age: Int,
    val email: String
)

val user1 = User("Ana", 20, "ana@mail.com")
val user2 = user1.copy(name = "Darwin")

// Resultado:
// user1 → User("Ana", 20, "ana@mail.com")      ← Original intacto
// user2 → User("Darwin", 20, "ana@mail.com")   ← Nuevo objeto

📋 Analogía: Fotocopiando un Documento

copy() es como hacer una fotocopia de un documento donde puedes cambiar algunas partes con corrector líquido antes de fotocopiar.

Original: Tu documento se mantiene intacto (user1)

Copia: Haces una fotocopia y modificas solo lo necesario (user2)

Ahora tienes dos documentos diferentes, el original y la versión modificada.

Actualizando el Estado Correctamente

Para que StateFlow emita un nuevo valor, debes crear un nuevo objeto. La forma más común es usar la función update() con copy():

// ✅ Forma correcta: Crea un nuevo objeto
_uiState.update {
    it.copy(name = "Darwin")
}

// ❌ Forma incorrecta: Modifica el objeto actual (NO emite)
_uiState.value.name = "Darwin"  // Esto no funcionará

🚦 Analogía: El Semáforo Inteligente

StateFlow es como un semáforo inteligente que solo cambia cuando detecta un auto nuevo (nueva referencia):

❌ No funciona: Si solo cambias el color del auto actual, el semáforo no lo detecta.

✅ Funciona: Cuando llega un auto diferente (aunque sea del mismo modelo pero nuevo), el semáforo cambia y todos lo ven.

Por eso necesitas copy(): para crear ese "auto nuevo" que active el semáforo.

📝 Resumen - Puntos Clave

  • StateFlow emite cambios solo cuando la referencia del objeto cambia
  • Usa copy() para crear un nuevo objeto con los cambios necesarios
  • La función update() es tu herramienta principal para actualizar el estado
  • Nunca modifiques las propiedades del objeto directamente, siempre crea uno nuevo
  • Mantén el MutableStateFlow privado y expone solo el StateFlow de lectura
🎯 Recuerda siempre: Para que StateFlow emita un nuevo estado, debes crear un nuevo objeto usando copy(). Sin un nuevo objeto, no hay emisión.
